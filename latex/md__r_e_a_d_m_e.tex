Цель работы — изучение основ создания документации с использованием разметки Markdown и управление версиями с помощью Git. В задачи лабораторной работы входит\+:


\begin{DoxyItemize}
\item Создание документации в формате Markdown
\item Инициализация и управление локальным репозиторием Git
\item Генерация документации с помощью Doxygen для кода на C++
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysection{\texorpdfstring{Шаги выполнения работы}{Шаги выполнения работы}}\label{md__r_e_a_d_m_e_autotoc_md2}

\begin{DoxyEnumerate}
\item Создайте файл документации в формате Markdown.
\item Добавьте в документ заголовки, списки, код и ссылки.
\item Инициализируйте локальный репозиторий Git.
\item Закоммитьте файл документации.
\item (Опционально) Свяжите репозиторий с Git\+Hub или Git\+Lab.
\item Используйте {\ttfamily git push} для отправки изменений в удаленный репозиторий.
\item Создайте простой проект на C++ с комментированием Doxygen.
\item Сгенерируйте документацию с помощью Doxygen.
